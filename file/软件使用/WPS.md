## WPS表格
#### 怎么根据某列数据的大小，把行内容作为一个整体，重新排序各行的位置
全选所在列 --> 打开【数据】选项卡 -->点击【排序】-->  勾选“扩展选定区域”‌ --> 选择升序或降序

注1：如果不勾选 **扩展选定区域**，那么只是对这一列的内容进行排序，而不会把行内容作为整体，对所在行排序。

注2：若表格第一行为标题行，‌需要同时勾选 **数据包含标题**
选中数据区域‌。未勾选“数据包含标题”会导致标题行被当作数据参与排序，造成信息丢失。

注3：**排序依据‌**默认为“数值”（适用于数字）或“文本”（适用于字母/汉字）。所以需要保持**数据一致性‌**，无混合类型（如数字与文本混杂），否则排序结果可能异常。

注4：排序前请**取消合并单元格**，否则可能导致排序失败或数据错乱。

<font color =red>注5：排序后如何恢复原始顺序‌？
在原表格增加一个【序号】列，用于记录初始的行顺序。排序后若需要还原，按此【序号】列升序即。</font>


#### 如何筛选出某一列包含指定内容的所有行
+ 对于单个数据或少量数据（采用手动点击的方式）
打开【开始】选项卡 -->点击【筛选】--> 在首行所有列单元格的右上角会出现一个可扩展的箭头 --> 点击即可看到该列所有包含的数据  --> 勾选目标数据 --> 表格会自动隐藏此列不是该数据的行 -->再次点击【筛选】则还原

+ 对于大量数据（采用公式匹配的方式）
**思路**：分为两步解决
	<font>1. 利用公式匹配，筛选出该列所有包含指定内容的行，并在该列旁生成新列用于标记，如果包含则输出1，不包含则输出0
	2. 对这个新列采用第一种筛选技巧，筛选出所有包含1的行即可
具体参考 [AI回答]( https://ima.qq.com/share/#/ai-chat/XyZ3aidrS) 

注1：在单元格中输入的公式 `=IF(COUNTIF($F$1:$F$6, A2)>0, "1", "0")`具体含义是：使用countif函数判断，A2单元格数据是否和从F1到F6中的某个数据相同；再使用if函数进行输出，如果包含输出1，不包含输出0.
> 当在某个输入该公式后，其他格可以直接自动填充。填充后，下一单元格中的公式会变成 `=IF(COUNTIF($F$1:$F$6, A3)>0, "1", "0")`，依此类推。其中使用`$`是为了固定后面的数字不会随着自动填充而改变。

注2：如何自动填充？
最好在需要检测的列旁新开一列，在新列的某个单元格输入公式。之后将鼠标移动到该单元格的右下角，会出现一个黑色的加粗十字符号，双击即可实现自动填充。

注3：WPS数字精度最大为15位，超过15位数字的数据（例如身份证号）会当作字符串来保存，此时如果再采用这种数值匹配的方式会出现问题。
解决方式是：**删掉多余的位数**，例如同省的身份证号头是一样的，可以通过【开始】>>【查找】>>【替换】，将需要删除的内容全部替换为空，之后再对短数据进行数值匹配。
> 如果采用加减某个数的方式来删除多余的位数，经检测无法得到正确的计算结果，因为数据本身处于溢出状态，再进行运算会导致异常。

### WPS启动
#### 如何关闭WPS自启动
> 系统级管理：禁止深层自启
	> <font>1. 任务管理器禁用启动项
	Ctrl+Shift+Esc 打开任务管理器 → 「启动」 选项卡 → 禁用所有 WPSOffice、WPSCloudService 等条目。
	>
	> <font>2. 停止后台服务 系统修复工具 
	Win+R 输入 services.msc → 找到 WPSCloudService 和 WPSOffice更新服务 → 右键属性 → 启动类型改为 “禁用”。
	>
	> <font>3. 清理计划任务
	Win+R 输入 taskschd.msc → 在任务计划程序库中删除所有 WPS相关任务（如自动更新、云同步）。

>高级清理：根除残留配置
	><font>1. 手动清除缓存
	Win+R 输入 %appdata%\Kingsoft\wps → 删除 cache 和 addons 文件夹内容，避免残留配置触发自启。
	><font>2. 注册表清理（谨慎操作）
	Win+R 输入 regedit → 定位路径：
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run
→ 删除含 “Kingsoft” 或 “WPS” 的键值（操作前备份注册表）。
